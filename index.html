Genomics Workshop

This hands-on workshop introduces students to a complete analysis pipeline for Next-Generation Sequencing (NGS) data generated using Illumina platforms, with a focus on population genetics.

The workshop primarily uses whole genome resequencing (WGS) data, but the same analytical framework can be adapted for RAD-seq or amplicon-seq datasets when a suitable reference genome is available.

By the end of the workshop, participants will understand how to:

  • Work with raw sequencing data (FASTQ files)
  • Perform quality control and preprocessing
  • Map reads to a reference genome
  • Call and filter genetic variants (SNPs/indels)
  • Explore population structure using genomic data

Pre-requisites & Setup

There are two ways to run this workshop. One option is through Google Colab, which allows users to run the entire pipeline in a cloud-based environment without installing software locally.

Alternatively, participants may run the workshop on their local machines using a Unix/Linux shell environment.

Windows users may install MobaXterm. Mac and Linux systems already provide a native terminal.

Workshop Pipeline

1. Initial Processing of FASTQ Files

Raw sequencing data often contains low-quality bases and technical artifacts. In this step, participants will assess read quality and remove sequencing adapters and low-quality regions to generate clean, analysis-ready reads.

  • FASTQ File Processing
  • Solutions File: FASTQ File Processing Solutions

Note for Google Colab users: The Colab notebook includes both read preprocessing (quality control and adapter trimming) and mapping of reads to the reference genome. The separate mapping step can therefore be skipped when following the Colab workflow.

2. Mapping Reads to Reference Genome

Cleaned sequencing reads are aligned to a reference genome to determine their genomic origin. The resulting alignment files form the foundation for variant discovery and downstream population genetic analyses.

  • Read Mapping and Sorting
  • Solutions File: Read Mapping and Sorting Solutions

3. Identifying Variants (SNPs/Indels)

Once reads are mapped, genetic variants such as single nucleotide polymorphisms (SNPs) and small insertions/deletions (indels) are identified. This step converts alignment data into variant call format (VCF) files.

  • Variant Identification

4. Filtering Variants

Raw variant calls often contain false positives and low-confidence sites. In this section, participants will apply quality, depth, and missingness filters to generate a high-confidence variant dataset.

  • Variant Filtering
  • Solutions File: Variant Filtering Solution
  • Explanation of filters: Variant Filter Descriptions

5. Population Structure & Visualization

The final section focuses on exploring genetic variation across individuals and populations. Participants will visualize population structure using approaches such as principal component analysis (PCA) and interpret biological patterns from genomic data.

  • Basic Population Genetics
